현재 사주앱 모바일 UI가 전체적으로 정보가 너무 한 번에 많이 노출되어서
사용자가 어디부터 봐야 하는지 헷갈립니다.

목표는 “정보량 축소”가 아니라
“정보 우선순위 재정리 + 단계적 노출 + 시각적 질서 회복”입니다.

아래 원칙대로 모바일 기준으로 UI 구조를 재정리해주세요.

[핵심 문제]
1. 카드/배지/강조박스가 너무 많아서 모든 정보가 중요해 보임
2. 섹션 간 위계가 약해서 화면이 길고 산만하게 느껴짐
3. 설명 텍스트, 요약, 분석, 태그가 동시에 보여 피로감이 큼
4. 컬러 사용이 많고 의미 체계가 불명확해 시선 분산이 큼
5. “바로 봐야 하는 핵심 정보”와 “궁금하면 펼쳐볼 상세 정보”가 분리되지 않음

[개선 목표]
- 첫 화면에서 사용자는 핵심 요약만 빠르게 이해할 수 있어야 함
- 상세 해석은 접기/펼치기(accordion)나 탭으로 단계적으로 노출
- 같은 레벨의 정보는 같은 UI 패턴으로 통일
- 컬러는 의미 있는 상태값에만 제한적으로 사용
- 긴 화면에서도 리듬감 있게 읽히도록 섹션 구조 단순화

[공통 UI 원칙]
1. 한 화면의 최상단에는 반드시 “한 줄 요약”과 “핵심 상태”만 노출
2. 상세 해석, 보충 설명, 부가 분석은 기본 접힘 상태
3. 카드 안에 카드가 들어가는 중첩 구조 최소화
4. 배지/칩은 정보 전달에 꼭 필요한 것만 남기고 2~3개 이하로 제한
5. 컬러는 브랜드 포인트 + 상태 강조용만 사용하고, 나머지는 중성 컬러 위주로 정리
6. 긴 설명문은 한 번에 다 보여주지 말고 요약 2~3줄 + 더보기 구조 사용
7. 모바일에서 각 섹션은 명확한 제목, 짧은 요약, 펼침 구조를 기본으로 설계
8. 시각적으로 가장 중요한 것은 큰 제목/점수/요약문이고, 나머지는 보조 계층으로 내려야 함

[화면별 개선 요청]

1) 홈 화면
- 오늘의 운 핵심 요약 카드만 가장 위에 크게 유지
- 영역별 운(관계운/재물운/건강운/일운)은 4개 전부 크게 노출하지 말고
  핵심 2개만 먼저 보이고 나머지는 접기 또는 슬라이드 구조로 변경
- 현재 운 흐름(대운/세운/일진)은 하나의 요약 카드로 통합
- 사용자가 홈에서 가장 먼저 봐야 하는 것은
  “오늘 전체 분위기 / 가장 좋은 영역 / 주의할 영역 / 오늘 일진” 정도로 제한

2) 내 사주 화면
- 현재 정보가 너무 많아 한 페이지에 사주 원국, 오행, 해석, 십신, 용신, 성격 분석, 기타 정보가 전부 나열됨
- 아래 구조로 재편:
  A. 상단 요약 영역
    - 이름 / 출생 정보 / 일주 / 한 줄 성향 요약
  B. 핵심 분석 탭
    - 탭1: 원국
    - 탭2: 성향
    - 탭3: 운세
    - 탭4: 해석
  C. 각 탭 내부도 “핵심 요약 > 상세 펼침” 구조 적용
- 오행 분포, 십성 분포, 용신/기신, 성향 해석은 한 번에 다 보이지 않게 접기 처리
- 사주 공부용 정보와 일반 사용자용 해석을 분리해서
  일반 사용자는 해석 중심으로, 전문 정보는 ‘자세히 보기’ 안에 넣기

3) 상대 화면
- 리스트 화면은 매우 단순하게 유지
- 상대 카드에는 이름 / 생년월일 / 핵심 태그 / CTA 2개(사주 보기, 궁합 보기)만 노출
- 수정/삭제 아이콘은 시각적으로 덜 튀게 정리
- 검색창과 등록 버튼의 위계를 명확히 하고, 빈 화면 UX도 설계

4) 궁합 화면
- 현재 가장 복잡한 화면이므로 반드시 단계적 구조로 재편
- 아래 순서로 단순화:
  A. 궁합 점수 + 한 줄 총평
  B. 핵심 궁합 포인트 3개
  C. 관계 장점 / 주의점
  D. 상세 분석(접기)
  E. 시기/운 흐름(접기)
- 명리 용어 기반 상세 해석은 기본 접힘 처리
- 사용자는 먼저 “그래서 이 관계가 어떤 관계인지”를 이해해야 하므로
  점수, 키워드, 총평, 장단점이 먼저 나와야 함
- 세부 원리 설명은 그 뒤에 배치

[디자인 시스템 정리]
- 텍스트 스타일 수를 줄여서 제목 / 섹션제목 / 본문 / 캡션 4단계 정도로 정리
- 카드 radius, padding, gap 기준 통일
- 칩/배지 스타일 통일
- divider와 여백으로 구획을 만들고, 컬러 박스 남발은 줄이기
- 한 섹션 안에서 강조 색상은 1개만 사용

[구체적으로 해줬으면 하는 작업]
1. 각 화면의 정보 우선순위 재정의
2. 중복되는 카드/섹션 통합
3. accordion / tab / summary 구조로 단계적 노출 구현
4. 모바일 기준 spacing과 typography 체계 정리
5. 기존 데이터는 유지하되 UI만 더 단정하고 덜 피곤하게 재배치

결과물은
- 홈 / 내 사주 / 상대 / 궁합
각 화면별 wireframe 수준 구조안 + 실제 UI 반영 코드까지 제안해주세요.
